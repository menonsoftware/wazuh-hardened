# Flow and Architecture Diagrams

This document provides detailed visual diagrams for the SOC demo stack.

## 1. System Architecture (Logical)

```mermaid
flowchart LR
  subgraph AGENTS[Agents Tier]
    A1[Wazuh Agents<br/>1k to 100k endpoints]
  end

  subgraph REGIONAL[Regional Tier - Core SIEM]
    M[Wazuh Manager Master<br/>1514/tcp, 1515/tcp, 55000/tcp]
    I1[Wazuh Indexer 1<br/>9200, 9300]
    I2[Wazuh Indexer 2<br/>9201, 9301]
  end

  subgraph CENTRAL[Central Tier - Visualization]
    SH[Wazuh Searchhead<br/>9200 internal]
    D[Wazuh Dashboard<br/>5601 internal HTTPS]
  end

  subgraph ZAMMAD[Helpdesk Tier]
    ZN[Zammad Nginx<br/>8443 internal HTTPS]
    ZR[Zammad Rails<br/>3000]
    ZW[Zammad WebSocket<br/>6042]
    ZWK[Zammad Worker]
    ZP[(PostgreSQL)]
    ZC[(Redis)]
  end

  subgraph EDGE[Edge Tier - Ingress]
    T[Traefik<br/>80, 443]
  end

  A1 -->|Event forwarding 1514/tcp| M
  M -->|Indexed security data| I1
  M -->|Indexed security data| I2

  D -->|OpenSearch queries| I1
  D -->|OpenSearch queries| I2
  D --- SH

  T -->|Host dashboard FQDN| D
  T -->|Host helpdesk FQDN| ZN

  ZN --> ZR
  ZN --> ZW
  ZR --> ZP
  ZR --> ZC
  ZW --> ZC
  ZWK --> ZC
```

## 3. Full Startup and First-Run Security Flow

```mermaid
flowchart TD
  S0[Run ./scripts/start_all.sh] --> S1{vm.max_map_count >= 262144?}
  S1 -- No --> E1[Exit with remediation message]
  S1 -- Yes --> S2[Create edge-net if missing]
  S2 --> S3[Start regional compose]
  S3 --> S4[Wait for wazuh.indexer.1 HTTPS readiness]
  S4 --> S5{regional/state/.security_initialized exists?}
  S5 -- No --> S6[Hash INDEXER_PASSWORD and update internal_users.yml]
  S6 --> S7[Run securityadmin.sh on indexer]
  S7 --> S8[Write regional sentinel]
  S5 -- Yes --> S9[Skip regional security bootstrap]
  S8 --> C1[Start central searchhead]
  S9 --> C1
  C1 --> C2[Wait for searchhead HTTPS readiness]
  C2 --> C3{central/state/.security_initialized exists?}
  C3 -- No --> C4[Hash INDEXER_PASSWORD and apply securityadmin.sh]
  C4 --> C5[Write central sentinel]
  C3 -- Yes --> C6[Skip central security bootstrap]
  C5 --> C7[Start central dashboard]
  C6 --> C7
  C7 --> Z1[Start zammad compose]
  Z1 --> E2[Start edge compose]
  E2 --> DONE[Stack startup complete]
```

## 4. Security Event to Ticket Flow

```mermaid
sequenceDiagram
  autonumber
  participant Agent as Wazuh Agent
  participant Manager as Wazuh Manager
  participant Indexer as Wazuh Indexer Cluster
  participant Dashboard as Wazuh Dashboard
  participant Analyst as SOC Analyst
  participant Zammad as Zammad Helpdesk

  Agent->>Manager: Send events and alerts (1514/tcp)
  Manager->>Indexer: Forward parsed and enriched events
  Dashboard->>Indexer: Query alerts and telemetry
  Analyst->>Dashboard: Triage and investigate findings
  Analyst->>Zammad: Create incident ticket with evidence
  Zammad-->>Analyst: Track status, assignment, and resolution
```

## 5. Certificate and TLS Dependency Flow

```mermaid
flowchart LR
  ROOTENV[Root .env subject values] --> SSL[./scripts/ssl.sh]
  SSL --> CERTS[certs/*.pem generated]
  CERTS --> REGIONALCFG[regional/opensearch.yml admin_dn and nodes_dn]
  CERTS --> CENTRALCFG[central/opensearch.yml admin_dn and nodes_dn]
  REGIONALCFG --> START[./scripts/start_all.sh]
  CENTRALCFG --> START
  START --> SECBOOT[securityadmin.sh initialization]
  SECBOOT --> SENTINELS[regional and central sentinel files]
```
