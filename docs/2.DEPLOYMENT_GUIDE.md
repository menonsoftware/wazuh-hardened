# Deployment Guide

This guide walks through a clean deployment of the SOC demo stack:

- Regional: Wazuh indexer cluster + Wazuh manager
- Central: Wazuh dashboard + searchhead
- Zammad: Ticketing stack
- Edge: Traefik ingress with ACME certificates

## 1. Architecture and Traffic Flow

High-level flow:

- Agents send events to Wazuh Manager on `1514/tcp`
- Manager forwards indexed data to regional indexers
- Dashboard queries indexers
- Traefik exposes browser-facing endpoints on `443`

Public endpoints:

- Dashboard: `https://<DASHBOARD_FQDN>`
- Helpdesk: `https://<HELPDESK_FQDN>`

For detailed visual diagrams, see `docs/5.FLOW_AND_ARCHITECTURE_DIAGRAMS.md`.

## 2. Host Prerequisites

Use a Linux host (Ubuntu 22.04+ recommended) with:

- Docker Engine 20.10+
- Docker Compose v2
- OpenSSL
- Sufficient resources (24 GB RAM minimum, 32 GB recommended)

Set OpenSearch kernel requirement:

```bash
sudo sysctl -w vm.max_map_count=262144
```

Persist after reboot:

```bash
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
```

Verify:

```bash
cat /proc/sys/vm/max_map_count
```

Expected: value is `262144` or higher.

## 3. DNS and FQDN Preparation

Ensure DNS records resolve to your host for:

- `DASHBOARD_FQDN`
- `HELPDESK_FQDN`

If this is a lab environment, temporary `/etc/hosts` entries are acceptable.

## 4. Configure Environment Files

The project uses multiple `.env` files. Update all before first start.

### 4.1 Root `.env` (certificate metadata)

File: `.env`

Required keys:

- `DOMAIN`
- `DAYS`
- `COUNTRY`
- `STATE`
- `LOCALITY`
- `ORGANIZATION`
- `CN_ROOT`
- `CN_ADMIN`

Important:

- Keep `ORGANIZATION` consistent with OpenSearch DNs expected by `regional/opensearch.yml` and `central/opensearch.yml`.
- If you change subject details, regenerate certificates with `./scripts/ssl.sh`.

### 4.2 Regional `.env`

File: `regional/.env`

Set:

- Cluster and domain: `REGIONAL_CLUSTER_NAME`, `DOMAIN`
- Indexer credentials: `INDEXER_USER`, `INDEXER_PASSWORD`, `OPENSEARCH_INITIAL_ADMIN_PASSWORD`
- Java and resources: `INDEXER_JAVA_OPTS`, CPU/memory limits and reservations
- Ports: `INDEXER_PORT_1`, `INDEXER_PORT_2`, `INDEXER_PORT_3`, `INDEXER_PORT_4`
- Manager resources and ports: `MANAGER_*`

### 4.3 Central `.env`

File: `central/.env`

Set:

- Cluster: `CENTRAL_CLUSTER_NAME`
- Dashboard routing: `DASHBOARD_FQDN`
- Shared credentials: `INDEXER_USER`, `INDEXER_PASSWORD`, `OPENSEARCH_INITIAL_ADMIN_PASSWORD`
- Searchhead and dashboard Java/resources: `SEARCHHEAD_*`, `DASHBOARD_*`

### 4.4 Zammad `.env`

File: `zammad/.env`

Set:

- Domain and route: `DOMAIN`, `HELPDESK_FQDN`
- Database credentials: `POSTGRES_USER`, `POSTGRES_PASSWORD`
- Resource limits/reservations for Postgres, Redis, Rails, WebSocket, Worker, Nginx

Security note:

- Use a strong random `POSTGRES_PASSWORD`.

### 4.5 Edge `.env`

File: `edge/.env`

Set:

- `TRAEFIK_ACME_EMAIL`
- Traefik CPU/memory limits and reservations

## 5. Generate TLS Certificates and Data Directories

Run from project root:

```bash
./scripts/ssl.sh
```

This script:

- Creates/refreshes `certs/`
- Generates root CA and node certificates
- Creates data and state directories under `regional/`, `central/`, and `zammad/`
- Applies container-friendly ownership (`1000:1000`) for data directories

Quick checks:

```bash
ls -1 certs | head
```

```bash
test -d regional/data/indexer1 && echo "regional indexer data dir exists"
```

## 6. Optional Image Build

If you need to rebuild component images:

```bash
./scripts/build_all.sh
```

If using published images only, you can skip this step.

## 7. Start the Full Stack

Run:

```bash
./scripts/start_all.sh
```

What happens in order:

1. Validates `vm.max_map_count`
2. Creates shared Docker network `edge-net` if missing
3. Starts components in order: `regional` -> `central` -> `zammad` -> `edge`
4. Performs first-run OpenSearch security initialization for:
   - `regional` (`regional/state/.security_initialized`)
   - `central` (`central/state/.security_initialized`)

First start can take several minutes while health checks stabilize.

## 8. Verify Deployment

### 8.1 Container status by component

```bash
cd regional && docker compose ps
cd ../central && docker compose ps
cd ../zammad && docker compose ps
cd ../edge && docker compose ps
```

Expected: all long-running services show `Up` and eventually `healthy` where health checks exist.

### 8.2 Verify security initialization sentinels

```bash
ls -la regional/state
ls -la central/state
```

Expected:

- `regional/state/.security_initialized`
- `central/state/.security_initialized`

### 8.3 Validate ingress routes

From browser:

- `https://<DASHBOARD_FQDN>`
- `https://<HELPDESK_FQDN>`

From shell:

```bash
curl -kI https://<DASHBOARD_FQDN>
curl -kI https://<HELPDESK_FQDN>
```

## 9. Post-Deployment Hardening Checklist

- Replace weak/default credentials in `.env` files
- Keep all configuration and certificate mounts read-only where applicable
- Restrict host firewall to required ports only (80/443, and manager ports as needed)
- Back up `certs/` and data directories regularly
- Review `opensearch.ssl.verificationMode` in `central/opensearch_dashboards.yml` (currently permissive)

## 10. Known Operational Notes

- Dashboard and helpdesk are exposed through Traefik on `edge-net`
- `start_all.sh` handles first-run OpenSearch security bootstrap automatically
- If certificate subject fields change, rerun `./scripts/ssl.sh` and reinitialize as needed
- Agent ingestion uses manager ports from `regional/.env` (`1514`, `1515`, `55000` mapped by variables)
