# Quickstart

Use this guide to bring up the SOC demo quickly on a fresh Linux host.

## 1. Host Precheck

Run:

```bash
sudo sysctl -w vm.max_map_count=262144
```

Optional persistence:

```bash
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
```

## 2. Set Required `.env` Values

Update these files before starting:

- `.env`
- `regional/.env`
- `central/.env`
- `zammad/.env`
- `edge/.env`

Minimum values to verify:

- Root cert metadata (`DOMAIN`, `ORGANIZATION`, etc.)
- Credentials (`INDEXER_PASSWORD`, `POSTGRES_PASSWORD`)
- Public routes (`DASHBOARD_FQDN`, `HELPDESK_FQDN`)
- ACME email (`TRAEFIK_ACME_EMAIL`)

## 3. Generate Certificates and Data Directories

```bash
./scripts/ssl.sh
```

## 4. Start the Full Stack

```bash
./scripts/start_all.sh
```

Startup order is automated:

- `regional` -> `central` -> `zammad` -> `edge`

## 5. Validate Service Health

```bash
cd regional && docker compose ps
cd ../central && docker compose ps
cd ../zammad && docker compose ps
cd ../edge && docker compose ps
```

Check sentinel files (first-run security initialized):

```bash
ls -la regional/state
ls -la central/state
```

## 6. Open the Applications

- Dashboard: `https://<DASHBOARD_FQDN>`
- Helpdesk: `https://<HELPDESK_FQDN>`

## 7. Common Control Commands

Stop all:

```bash
./scripts/stop_all.sh
```

Backup:

```bash
./scripts/backup.sh
```

Full reset (destructive):

```bash
./scripts/erase.sh
```
